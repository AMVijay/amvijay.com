How HTTPS Works?

HTTPS is secured http protocol to transmit data between client and server. It uses SSL (secured socket layer) / TLS (transport layer security) . HTTPS uses asymmetric keys for encryption and decryption. SSL certificate which includes asymmetric key (private key, public key) is required for https to work.

SSL certificate has two part of it, those are:

  • Public key
    • This is the key exposed to client. Browser client gets this key from server and confirms server identity.
  • Private key
    • This is key used server not exposed to anybody. Server uses this key to decrypt the message received from client during handshake.

  • Step1: browser client sends hello message http request to server along with ssl/tls versions it can communicate.

  • Step2: server responds back to browser client with hello message , public key and the highest grade ssl/tls version among the one sent by client .
  • Step 3: client validates the ssl certificate for identity and authenticity. Then, browser client generates new symmetric key. It is sent to server as encrypted using public key.
  • Step 4: then, server decrypts the symmetric key using its own private key. Then, it sends back encrypted message to client using the symmetric key.
  • Step 5 : here onwards, browser client sends all the follow up requests using the symmetric key sent to server during handshake.
